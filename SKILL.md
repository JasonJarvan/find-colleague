# find-colleague

根据**项目**反查**负责的同事**（也支持同事→项目、团队→人）。事实源是各团队周报，
抽取归一成 `data/colleague-project.md`。本 Skill 是 v1（Markdown + LLM）形态。

## 何时用

- 「<项目> 该找谁 / 谁在做 <项目> / 谁负责 <模块>」→ 走 **查询**
- 「<同事> 在做哪些项目」「<团队> 有哪些人」→ 走 **查询**
- 「<同事> 是什么职位 / <团队> 都有谁、各做什么」（**职位/人物**维度）→ 走 **查询**，
  按团队分组逐人答「姓名（团队）｜ 职位 ｜ 在做的项目」
- 「更新一下数据 / 重新抓周报 / 数据过期了」→ 走 **刷新**

> v2 CLI 已把上述查询固化为命令，可作事实源：`find-colleague who/query/projects` 查人-项目，
> `find-colleague people [--team … | --name …]` 逐人列「姓名（团队）｜ 职位 ｜ 项目→工作清单」。
> 本 Skill（v1）仍以 `data/colleague-project.md` 为查询对象。

## 能力一：查询（默认）

纯 LLM 检索，**只读** `data/colleague-project.md`，不读 `data/raw/`。

0. **先确认提问者是谁**（找人是为了找"别人"，这一步不能省）：
   - 从会话的 `userEmail` 推断姓名（如 `your.name@company.example` → 张三），在映射表
     § 主映射表里能对上同事名就认定；
   - 推断不出、或映射表里没有这个人时，**直接问用户「你是谁/你在哪个团队」**再继续。
1. 读 `data/colleague-project.md`。
2. 归一用户的查询词：先查 § 项目别名表，把「云平台V2 / OME」等异写映射到归一项目名；
   团队词（产品/工程/运营/算法）对应 § 主映射表的 section。
3. 按查询方向匹配：
   - **项目 → 同事**：优先用 § 按项目反查索引；若是别名表未覆盖的模糊词，回退到主映射表
     按「项目」「角色/做了什么」列做语义匹配。
   - **同事 → 项目**：在主映射表按「同事」列汇总该人所有行。
   - **团队 → 人**：取对应 `### section` 下的同事去重。
4. **排除提问者本人**：结果里去掉步骤 0 认定的提问者。若提问者本人正是该项目的主力/负责人，
   明确点出「你自己就是这块的主力」，然后给**可对接的协作方 / 相邻 owner / 上下游**，
   而不是把他指回自己。
5. 返回：命中的同事名 + 所属团队（来自 section）+ 一句「做了什么」+ 来源页缩写。
   命中多人时按相关度排序，并说明判断依据（来自哪条记录）。
6. **诚实边界**：只回答数据里有的；周报未覆盖的项目/人直说「数据中未见」，
   不要臆测。必要时建议先**刷新**。

## 能力二：刷新（按需，会读外部）

重抓周报并重建映射表。**有外部读取副作用**，执行前向用户确认。

1. 读 `data/sources.md`，取「适用版本 = v1」的页面（**只这两个，不要遍历目录**）。
2. 对 step 1 取到的每个页面，经 Atlassian MCP 读取
   （`mcp__claude_ai_Atlassian_Rovo__getConfluencePage`，`contentFormat=markdown`）。
   **cloudId / space / page_id 全部取自 `data/sources.md` 对应行（gitignored，含真实值）——
   本文件不重复、不硬编码这些值（单一事实源 = sources.md）。**
3. 先把原文落 `data/raw/<team>-<period>.md` 快照（覆盖旧快照），再抽取——保证可重跑、可追溯。
4. 按 `data/colleague-project.md` 的既有结构重建：①团队分类 ②项目别名表 ③主映射表（按
   团队 section）④按项目反查索引。抽取规则见该文件头注释与本 Skill § 抽取与归一约定。
5. 更新 `data/sources.md` 的「抓取记录」与 `colleague-project.md` 的 `last_refreshed_at`。
6. 鉴权失效（headless/cron 常见）时停下来报告，不要伪造数据。

## 抽取与归一约定

- **拆行**：同一人同一周涉及多个项目 → 拆成多行（一行一个 (同事, 项目)）。
- **项目归一**：同名异写合并到归一项目名，新映射补进 § 项目别名表（不另建文件）。
- **团队**：是「人」的属性，四个**并列**团队 `产品（含设计）/ 工程 / 运营 / 算法`，
  用主映射表的 `### section` 表示；与「来源页」解耦（来源 space ≠ 团队划分，需按职能判断）。
  设计职能并入产品，行内可标 `(设计)`。
- **粒度**：抽「可识别的项目/模块」而非每条琐碎任务；「角色/做了什么」一句话。
- **模糊归类**：人-团队归属拿不准时，标注并询问用户，不要自行硬归。

## 数据契约

| 文件 | 角色 |
|---|---|
| `data/colleague-project.md` | ★ 查询唯一对象（归一映射表） |
| `data/sources.md` | 周报来源登记 + 抓取记录 |
| `data/raw/*.md` | 原文快照（provenance / 重抽取依据，不作查询对象） |

## 边界与演进

- v1 只覆盖 `data/sources.md` 里的 v1 两个页；不遍历目录、不做语义向量检索。
- 数据膨胀 / 需定期爬取多来源 / 需服务化 → 升级 v2（Service + MCP，SQLite + 向量）。
  升级触发与整体设计见仓库根 `CLAUDE.md`。
